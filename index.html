Workflow

7-step automated analysis pipeline

From raw AMM documents to actionable engineering reports — fully automated.

01

INTAKE

Accepts plain text, single PDF, or multi-chapter PDF packages. Auto-detects aircraft type, revision number, and date.

02

PARSE

Extracts TASK blocks, LEP status, HIGHLIGHTS section, intervals, materials, tools, special tests, and man-hours using ATA iSpec 2200 patterns.

03

DIFF

Matches tasks by task_ref and fuzzy title. Compares intervals (including TIME LIMITS tables), materials, tools, special tests, and man-hours.

04

CLASSIFY

Tags every change with one of 14 change types and assigns HIGH / MEDIUM / LOW priority. INTERVAL_REDUCED is always HIGH.

05

FILTER

Applies MRO or CAMO context filter. MRO focuses on tools/materials/tests; CAMO focuses on interval changes vs. AMP.

06

REPORT

Generates a formatted Excel workbook (6 sheets) + Markdown summary. Color-coded rows, auto-filters, and freeze panes included.

07

RECOMMEND

Every change gets a human-readable action recommendation — from AMP amendment notices to borescope crew scheduling.

Dual Context

Built for both MRO and CAMO engineers

Same analysis engine, two tailored output views.

🔧

MRO Mode

Maintenance Planning Engineers

Everything you need before a C-check: new tools, materials, special tests, and man-hour impacts.

  • New special tools & GSE required
  • Material / consumable spec changes
  • Borescope / NDT crew scheduling
  • Man-hour delta for work packages
  • New tasks added to check scope
  • Tasks removed from check packages
📋

CAMO Mode

Airworthiness / AMP Engineers

Full AMP impact matrix with authority notification flags. Match every interval change to your AMP task number.

  • Interval reductions flagged → AMP amendment
  • Fuzzy AMP task matching (task_ref + description)
  • Authority notification YES / EVALUATE flags
  • New tasks for AMP incorporation evaluation
  • Deleted tasks for AMP removal
  • AMP impact matrix as a dedicated Excel sheet

CLI Usage

Simple command-line interface

Four commands cover every use case.

Basic delta — two text files
amm-engineer analyze \ --old amm_rev15.txt \ --new amm_rev21.txt \ --aircraft "Cessna 172" \ --output-dir ./reports/
CAMO mode — with AMP matching
amm-engineer analyze \ --old amm_rev15.txt \ --new amm_rev21.txt \ --amp sample_AMP.xlsx \ --mode camo --aircraft "Cessna 172"
PDF input
amm-engineer analyze \ --old-pdf amm_old.pdf \ --new-pdf amm_new.pdf \ --aircraft "B737-800"
Extract only — single revision
amm-engineer extract \ --input amm_rev21.txt \ --aircraft "B737-800"
